1. ログイン方法

アクセス手順

  1. 1ブラウザで https://highalt-dashboard.vercel.app にアクセスします
  2. 2ログイン画面が表示されます
  3. 3管理者から発行された ユーザー名 と パスワード を入力します
  4. 4「ログイン」ボタンをクリックします

ユーザーの種類

ロール説明権限
管理者（admin）全店舗の閲覧・管理が可能全機能にアクセス可能
店長（store_manager）担当店舗の管理が可能自店舗のデータアップロード・閲覧、他店舗は閲覧のみ

セキュリティに関する注意

ログイン試行回数の制限: 5回連続でログインに失敗すると、15分間アカウントがロックされます。
セッションの有効期限: ログイン後24時間が経過すると自動的にログアウトされます。
パスワードは定期的に変更してください。

2. ダッシュボード画面の見方

ダッシュボードは店舗のPL（損益計算書）データを可視化する画面です。

2.1 期間選択

画面上部の期間選択エリアで、表示するデータの範囲を指定します。

  • 年: 対象年度を選択します（例: 2026年）
  • 月: 対象月を選択します（例: 4月）
  • 年間表示モードに切り替えることで、年度全体の推移を確認できます

2.2 店舗選択

  • ドロップダウンから対象店舗を選択します
  • 管理者は全7店舗（東日本橋、春日、船橋、巣鴨、祖師ヶ谷大蔵、下北沢、中目黒）から選択可能
  • 店長は自店舗がデフォルトで選択されます
  • 「本部（除外）」はPL計算の対象外です

2.3 KPIカード

画面上部に主要な経営指標がカード形式で表示されます。

2.4 グラフ

KPIカードの下に、データの推移や内訳を示すグラフが表示されます。

  • 月次推移グラフ: 売上・人件費・経費の月ごとの推移
  • 経費内訳: 勘定科目別の経費割合
  • 店舗比較: 全店舗の横並び比較（管理者のみ）

2.5 店舗比較画面

管理者は店舗比較画面で、全店舗のKPIを一覧で比較できます。

  • 売上、人件費、経費、営業利益を店舗横並びで表示
  • 前月比や前年同月比の変動も確認可能

3. アップロード画面

各種CSVファイルをアップロードしてデータを取り込みます。

3.1 共通の注意事項

対応形式: CSV（.csv）、Excel（.xlsx, .xls）
ファイルサイズ上限: 10MB
上書き: 既存データがある月にアップロードすると上書きされます（確認ダイアログあり）
アップロード履歴は自動的に記録されます。

3.2 人件費CSV

アップロード権限: 管理者のみ
  1. 1「アップロード」画面を開きます
  2. 2データ種別で「人件費」を選択します
  3. 3対象の年・月を指定します
  4. 4クラウド給与から出力した 支給控除一覧表（CSV/Excel）をアップロードします
  5. 5プレビュー画面で内容を確認し、「保存」をクリックします
自動処理:
従業員番号の千の位で店舗を自動判定（1xxx=東日本橋、2xxx=春日、...）
未登録の従業員がいる場合、店舗選択のアラートが表示されます。

3.3 経費CSV

経費データは2ステップでアップロードします。

ステップ1: Amazon注文履歴（オプション）

  1. 1Amazon注文履歴CSVを選択・アップロードします
  2. 2配送先住所から店舗が自動判定されます
  3. 3商品ごとの勘定科目が自動分類されます

ステップ2: PayPay銀行CSV

  1. 1PayPay銀行から出力した取引明細CSVをアップロードします
  2. 2システムが自動的に勘定科目の自動分類、AMAZON行のマッチング、収入取引の判定を処理します
  3. 3プレビュー画面で未分類の取引に手動で勘定科目を設定します
  4. 4手動で設定した勘定科目は 自動的にルールとして保存 され、次回以降は自動分類されます
  5. 5「保存」をクリックしてデータを確定します

3.4 売上CSV

  1. 1「売上」タブを選択します
  2. 2サブタブ（会員ML001 / 売上明細PL001 / 月次サマリMA002）を選択します
  3. 3対象店舗を指定します（年・月はCSVから自動認識されます）
  4. 4CSVファイルをアップロードします
  5. 5プレビューを確認後、保存します

3.5 予算CSV

  1. 1「予算」を選択します
  2. 2対象の年・月・店舗を指定します
  3. 3予算データCSVをアップロードします
  4. 4プレビューを確認後、保存します

4. 設定画面

管理者のみアクセス可能です。

4.1 従業員マッピング

従業員番号と所属店舗の対応関係を管理します。

  • 一覧表示: 登録済みの従業員マッピングが表示されます
  • 自動登録: 人件費CSVアップロード時に未登録の従業員が検出された場合、自動的に登録されます
  • 手動編集: 店舗の割り当てを手動で変更できます

4.2 経費分類ルール

PayPay銀行CSVの取引摘要に含まれるキーワードと勘定科目の対応ルールを管理します。

  • 一覧表示: 登録済みのルールが表示されます
  • 新規追加: キーワードと勘定科目を指定してルールを追加できます
  • 自動学習: 経費アップロード時に手動分類した結果が自動的にルールとして保存されます
  • 削除: 不要なルールを削除できます

4.3 Amazon商品マスタ

Amazon注文履歴の商品と勘定科目の対応を管理します。

  • 一覧表示: 登録済みのASIN・商品名・勘定科目が表示されます
  • 検索: ASINまたは商品名で検索できます
  • 新規追加: ASIN、商品名、勘定科目を指定して登録します
  • CSV一括取込: 商品マスタCSVを一括でインポートできます
  • 削除: 不要な商品マスタを削除できます

4.4 ユーザー管理

管理者のみアクセス可能です。

  • ユーザー一覧: 登録済みのユーザーが表示されます
  • 新規追加: ユーザー名、パスワード、ロール（管理者/店長）、担当店舗を指定して作成します
  • 編集: 既存ユーザーのロールや担当店舗を変更できます
  • 削除: ユーザーを削除できます

6. FAQ（よくある質問）

ログインできない場合

Qパスワードを忘れた場合はどうすればよいですか？
管理者に連絡してパスワードのリセットを依頼してください。管理者がユーザー管理画面からパスワードを再設定できます。
Q「ログイン試行回数が上限に達しました」と表示されます
セキュリティのため、5回連続でログインに失敗するとアカウントが15分間ロックされます。15分後に再度お試しください。

データアップロードについて

QCSVファイルのエンコーディングはどれを使えばよいですか？
PayPay銀行CSVはShift_JIS（cp932）形式で出力されますが、システムが自動判定しますのでそのままアップロードしてください。UTF-8のCSVも対応しています。
Q同じ月のデータを再度アップロードできますか？
はい、可能です。既存データがある場合は上書き確認のダイアログが表示されます。「はい」を選択すると既存データが新しいデータに置き換わります。
Qアップロードしたデータを削除できますか？
同じ月に空のデータを上書きアップロードすることで実質的に削除できます。完全な削除が必要な場合は管理者にご相談ください。
Qファイルサイズが10MBを超えてアップロードできません
CSVファイルの場合、不要な列やヘッダー行を削除してファイルサイズを削減してください。それでも解決しない場合は管理者にご相談ください。

ダッシュボードについて

Qダッシュボードにデータが表示されません
以下を確認してください:
  1. 正しい年・月・店舗が選択されているか
  2. 対象期間のデータがアップロード済みか
  3. ブラウザのキャッシュをクリアして再読み込みしてみてください
Q人件費の内訳が表示されません
店長アカウントでは、他店舗の人件費内訳は閲覧できません。自店舗の内訳のみ表示されます。全店舗の内訳を確認するには管理者アカウントでログインしてください。
Q経費の勘定科目が「未分類」のままです
経費アップロード時のプレビュー画面で手動分類するか、設定画面の「経費分類ルール」で該当キーワードのルールを追加してください。一度手動分類すると、次回以降は自動分類されます。

その他

Q複数のブラウザやデバイスから同時にログインできますか？
はい、可能です。ただし、それぞれのセッションは独立しており、一方でログアウトしても他方には影響しません。
Q対応しているブラウザは何ですか？
Google Chrome、Mozilla Firefox、Microsoft Edge、Safariの最新版に対応しています。Internet Explorerには対応していません。
Q操作中にエラーが発生した場合はどうすればよいですか？
ブラウザを再読み込みして再度お試しください。問題が解決しない場合は、エラーメッセージのスクリーンショットとともに管理者にご連絡ください。
